# Workspace

## Сама структура
Когда мы делаем init, в текущей папке создаётся:

root/
  .peer/
    obj/
    meta/

- .peer это служебная папка, скрытая для пользователя(вроде бы примерно как .git)

- obj сюда кладём копии файлов для скачивания 

- meta сюда потом пойдёт база данных  

В самой рабочей директории могут лежать:
- обычные локальные файлы
- *.empty пустые файлы(просто названия фаликов из сети)

## у Workspace есть поля(по сути просто пути до нужных папок)
- root
- .peer
- obj
- meta

## init
Workspace::init(path)

Создаёт новую структуру .peer.

Если .peer уже существует в этой папке или выше по дереву, то кидается исключение.
## discover
Workspace::discover(path)

Ищет .peer, поднимаясь вверх по папкам.

Работает примерно как git.

Если не находит .peer, кидает исключение.
## create_objectfile
Берёт локальный файл и копирует его в: .peer/obj/<object_id>
Пока object_id это просто имя файла.

Возвращает путь к созданному объекту.

Если файл не существует или это не обычный файл, кидается исключение.
## delete_objectfile
Удаляет объект из .peer/obj.

Если такого файла нет, кидается исключение.
## create_all_empty_files
Создаёт .empty файлы в root.

Например: ("a.txt" превращается в a.txt.empty(по хорошему бы исправить?))

Файл создаётся пустым и read only.

Используется, когда по метаданным мы знаем,
что файл есть, но локально он ещё не скачан.
