# Pear-to-Pear (p2p storage) — MVP

Идея проекта: p2p-сервис для хранения набора файлов на разных устройствах.  
В MVP — **линейная структура без папок**, базовые операции: **добавить / удалить / скачать**, интерфейс — **терминал**.  
Метаданные синхронизируются между устройствами, сами файлы хранятся на устройствах участников.

---

## Что лежит в репозитории

- `docs/` — **структура и материалы проекта** (диздок / описание MVP / заметки)  
  > Здесь зафиксировано, что именно мы реализуем и как это задумано.

- `examples/` — **пробный код / эксперименты**  
  > Небольшие тестовые реализации для работы с gRPC / protobuf и другими технологиями.

- `src/` — **основной код проекта**
  - `main.cpp` — точка входа CLI-приложения
  - `pear/` — библиотека проекта:
    - `fs/` — файловый слой
    - `db/` — слой работы с SQLite
    - `net/` — сетевой слой (gRPC)
    - `cli/` — логика CLI-команд

- `tests/` — модульные тесты (gTest)

## Технологии (MVP)

- Сеть: gRPC  
- Протокол: Protobuf  
- БД: SQLite  
- Файловый слой: `std::filesystem`  
- CLI: CLI11  
- Тесты: gTest
